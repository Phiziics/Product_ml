# Firefox Growth Intelligence

A product data science and experimentation project that analyzes browser growth, retention, A/B testing, causal impact, forecasting, and performance-risk monitoring using Python, SQL, Streamlit, and Docker.

## 1. Project Objective

Firefox wants to grow active usage while protecting product quality, privacy, and user trust.

This project simulates how a product data scientist would help a browser product team answer:

- Are active usage and engagement growing?
- Which users are most likely to return?
- Did a product experiment improve retention?
- Did the experiment hurt any guardrail metrics?
- Can we forecast future usage?
- Are performance anomalies creating product risk?

The project combines product analytics, experimentation, causal inference, machine learning, time series forecasting, anomaly detection, dashboarding, and containerized deployment.

## 2. Why This Project Matters

Modern product data science is not only about building models. A strong product data scientist must define useful metrics, design experiments, evaluate product changes, explain business impact, and communicate recommendations clearly.

This project is designed to show that workflow end to end:

```text
Raw data
→ data cleaning
→ product metric design
→ A/B testing
→ causal inference
→ retention modeling
→ usage forecasting
→ anomaly detection
→ dashboard
→ executive recommendations
